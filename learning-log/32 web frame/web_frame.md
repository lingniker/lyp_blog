## 前端框架

|-ui
  |- build  -------------------------------------------  用于打包的文件
  |- examples -----------------------------------------  展示的例子
  |- lib  ---------------------------------------------  生成的库
  |- packages -----------------------------------------  组件的页面
  |- src ----------------------------------------------  资源文件
  |- test ---------------------------------------------  测试的代码
  |- types --------------------------------------------  typescript

### 说下为什么这样设置
#### 其实这是 element 的结构目录是这样的。 我也参看了 at-ui. 和 ivews 的框架。大同小异，确切来说是 对 element 更加熟悉。所以这样设计了。
